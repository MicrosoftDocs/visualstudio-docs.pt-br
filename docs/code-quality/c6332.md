# <a name="c6332"></a>C6332
C6332 de aviso: Parâmetro inválido: passando zero como o parâmetro dwFreeType para \<function > não é permitido. Isso resulta na falha dessa chamada

 Esse aviso indica que um parâmetro inválido está sendo passado para VirtualFree ou VirtualFreeEx. VirtualFree e VirtualFreeEx rejeitam um parâmetro dwFreeType de zero. O parâmetro dwFreeType pode ser MEM_DECOMMIT ou MEM_RELEASE. No entanto, os valores MEM_DECOMMIT e MEM_RELEASE não podem ser usados juntos na mesma chamada. Além disso, certifique-se de que o valor de retorno da função VirtualFree não seja ignorado.

## <a name="example"></a>Exemplo
 O código a seguir gera esse aviso porque um parâmetro inválido é passado para a função VirtualFree:

```
#include <windows.h>
#define PAGELIMIT 80

DWORD dwPages = 0;  // count of pages
DWORD dwPageSize;   // page size

VOID f( VOID )
{
  LPVOID lpvBase;            // base address of the test memory
  BOOL bSuccess;
  SYSTEM_INFO sSysInfo;      // system information

  GetSystemInfo( &sSysInfo );
  dwPageSize = sSysInfo.dwPageSize;

  // Reserve pages in the process's virtual address space
  lpvBase = VirtualAlloc(
                         NULL,                // system selects address
                         PAGELIMIT*dwPageSize, // size of allocation
                         MEM_RESERVE,
                         PAGE_NOACCESS );
  if (lpvBase)
  {
    // code to access memory
  }
  else
  {
    return;
  }

  bSuccess = VirtualFree( lpvBase, 0, 0 );
  // code ...
}
```

 Para corrigir esse aviso, modifique a chamada para a função VirtualFree, conforme mostrado no código a seguir:

```
#include <windows.h>
#define PAGELIMIT 80

DWORD dwPages = 0;  // count of pages
DWORD dwPageSize;   // page size

VOID f( VOID )
{
  LPVOID lpvBase;            // base address of the test memory
  BOOL bSuccess;
  SYSTEM_INFO sSysInfo;      // system information

  GetSystemInfo( &sSysInfo );
  dwPageSize = sSysInfo.dwPageSize;

  // Reserve pages in the process's virtual address space
  lpvBase = VirtualAlloc(
                         NULL,                // system selects address
                         PAGELIMIT*dwPageSize, // size of allocation
                         MEM_RESERVE,
                         PAGE_NOACCESS );
  if (lpvBase)
  {
    // code to access memory
  }
  else
  {
    return;
  }

  bSuccess = VirtualFree( lpvBase, 0, MEM_RELEASE );
  // code ...
}
```

 O uso de VirtualAlloc e VirtualFree tem muitas armadilhas em termos de vazamentos de memória e exceções. Para evitar esses tipos de vazamentos e problemas de exceção totalmente, use os mecanismos fornecidos pela STL ( C++ Standard Template Library). Isso inclui [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)e [vector](/cpp/standard-library/vector). Para obter mais informações, consulte [apontadores inteligentes](/cpp/cpp/smart-pointers-modern-cpp) e [ C++ biblioteca padrão](/cpp/standard-library/cpp-standard-library-reference).
