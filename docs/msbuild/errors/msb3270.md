# <a name="msb3270-there-was-a-mismatch-between-the-processor-architecture-of-the-project-being-built-arch-and-the-processor-architecture-of-the-reference-reference-arch"></a>MSB3270: houve uma incompatibilidade entre a arquitetura do processador do projeto que está sendo criado ' Arch ' e a arquitetura do processador da referência ' reference ', ' Arch '

A mensagem completa é semelhante à seguinte:

```output
MSB3270: There was a mismatch between the processor architecture of the project being built 'architecture' and the processor architecture of the reference 'dependency', 'architecture'. This mismatch may cause runtime failures. Please consider changing the targeted processor architecture of your project through the Configuration Manager so as to align the processor architectures between your project and references, or take a dependency on references with a processor architecture that matches the targeted processor architecture of your project.
```

## <a name="possible-causes"></a>Possíveis causas

A mensagem diz que você está tentando usar uma dependência que é de uma arquitetura que não corresponde à arquitetura de destino para a qual seu projeto está configurado. Primeiro, você precisa confirmar se as arquiteturas de destino estão corretas e, em seguida, certificar-se de que todas as dependências referenciadas correspondam à arquitetura de destino. 

No Visual Studio, você pode usar o Configuration Manager para controlar quais arquiteturas são criadas.

No MSBuild, a `PlatformTarget` propriedade é usada para definir a plataforma de destino.
