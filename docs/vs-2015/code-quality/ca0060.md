# <a name="ca0060"></a>CA0060

Não é possível carregar assemblies referenciados indiretamente.

Este aviso ocorre se a ferramenta de análise não puder carregar assemblies indiretamente referenciados pelo assembly em análise. Uma "referência indireta" se refere a um assembly de referência que uma das referências de assembly do seu assembly de análise se refere ao. Por exemplo se a análise de código (FxCop) está analisando o assembly A e o assembly um faz referência ao assembly B e o assembly B faz referência ao assembly C, mas um não faz referência a assembly C, em seguida, o assembly C é uma referência indireta e assembly B é uma referência direta.

Isso pode causar erro CA0001 em outros lugares, como muitos estados inesperados podem ocorrer se isso ocorrer. Se você estiver usando a ferramenta de análise de código gerenciado dentro do Visual Studio, a seguir está alguns dos motivos que podem causar o aviso CA0060:

-   O assembly de entrada foi substituído e agora o arquivo de projeto aponta para um assembly que faz referência a outros assemblies fora da lista de referências no Visual Studio.

-   Se um projeto é modificado para invocar uma etapa de compilação personalizada antes FxCopCmd.exe é executado e o processo de compilação personalizado adiciona mais referências ao assembly de entrada e CA0060 de aviso será gerado.

-   No C++, você pode referenciar assemblies usando ForcedUsing em VCPROJ, substituindo a linha de comando para passar /FU ou adicionar #using \<Some.dll > no código-fonte somente.

-   Um terceiro assembly que tem algumas referências privadas para outros assemblies que não é necessário para compilar e executar seu código.

Em todos os casos, a resolução para esse problema é adicionar a referência ausente ao projeto do Visual Studio em si.
