# <a name="c6101"></a>C6101
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

aviso C6101: retornando memória não inicializada  
  
 Um caminho bem-sucedido por meio da função não define o `_Out_` parâmetro nomeado. Essa mensagem é gerada com base nas anotações SAL que indicam que a função em questão sempre terá sucesso. Uma função que não retorna uma indicação de êxito/falha deve definir todos os seus `_Out_` parâmetros porque o analisador supõe que o `_Out_` parâmetro é dados não inicializados antes que a função seja chamada e que a função definirá o parâmetro para que ele não seja mais não inicializado. Se a função indicar êxito/falha, o `_Out_` parâmetro não precisará ser definido no caso de falha e você poderá detectar e evitar o local não inicializado. Em ambos os casos, o objetivo é evitar a leitura de um local não inicializado. Se a função às vezes não toca em um `_Out_` parâmetro que é usado posteriormente, o parâmetro deve ser inicializado antes da chamada de função e ser marcado com a `_Inout_` anotação, ou o mais explícito `_Pre_null_` ou `_Pre_satisfies_()` quando apropriado. "Êxito parcial" pode ser manipulado com a `_When_` anotação. Para obter mais informações, consulte [usando anotações de sal para reduzir os defeitos de código C/C++](../code-quality/using-sal-annotations-to-reduce-c-cpp-code-defects.md).
