# <a name="receive-activity-designer"></a>Recebe o designer de atividades
O **designer de atividades Receber** é <xref:System.ServiceModel.Activities.Receive> usado para criar e configurar uma atividade. Uma atividade de <xref:System.ServiceModel.Activities.Receive> é uma atividade que receberá uma mensagem que pode ser um tipo interno como <xref:System.ServiceModel.Channels.Message>, <xref:System.IO.Stream> ou <xref:System.Xml.Linq.XElement>, ou um contrato definido de dados, o contrato de mensagem, ou a classe XML que pode ser serializada.

## <a name="the-receive-activity"></a>A atividade de receptor
 A atividade de <xref:System.ServiceModel.Activities.Receive> pode receber um item único ou vários itens dependendo do tipo de recebem o conteúdo usado. Uma atividade de <xref:System.ServiceModel.Activities.SendReply> pode ser associada a uma atividade de <xref:System.ServiceModel.Activities.Receive> que receberá uma mensagem como parte de um padrão de troca de solicitação/resposta de mensagem no serviço.

### <a name="using-the-receive-activity-designer"></a>Usando o designer de atividade de receptor
 O designer **de atividades Receber** pode ser encontrado na categoria **Mensagens** da **Caixa**de [!INCLUDE[wfd2](../includes/wfd2-md.md)] Ferramentas , que é acessada clicando na guia Caixa de **ferramentas** na (Alternativamente, selecione Barra de **ferramentas** no menu **Exibir** ou CTRL+ALT+X.)

 O designer de **atividades Receber** pode ser arrastado da [!INCLUDE[wfd2](../includes/wfd2-md.md)] caixa de **ferramentas** e jogado sobre a superfície onde as atividades geralmente são colocadas. Isso cria uma atividade de <xref:System.ServiceModel.Activities.Receive> com <xref:System.Activities.Activity.DisplayName%2A> padrão Receive. O <xref:System.Activities.Activity.DisplayName%2A> pode ser editado no cabeçalho do designer de **atividades Receber** ou na caixa **DisplayName** da grade de propriedade.

 Para criar <xref:System.ServiceModel.Activities.SendReply> uma atividade e <xref:System.ServiceModel.Activities.Receive> vinculá-la à atividade selecionada, clique com o botão direito do mouse no designer **de atividades 'Receber',** clique no item **Criar EnviarResposta** no menu de contexto' e o designer **SendReplyToReceive** aparece abaixo do designer **Receber.** A atividade de <xref:System.ServiceModel.Activities.SendReply> é uma atividade que envia a mensagem de resposta como parte de um padrão de troca de solicitação/resposta de mensagem no serviço. Ele pode ser configurado com o designer **SendReplyToReceive.**

 Alternativamente, o designer **de modelos ReceiveAndSendReply** na categoria **Mensagens** da **Caixa** de <xref:System.ServiceModel.Activities.Receive> Ferramentas <xref:System.ServiceModel.Activities.SendReply> pode ser usado para criar um par de atividades pré-configuradas. [!INCLUDE[crabout](../includes/crabout-md.md)]o uso do modelo **ReceiveAndSendReply** e **SendReplyToReceive,** consulte o tópico [ReceiveAndSendReply.](../workflow-designer/receiveandsendreply-template-designer.md)
