# <a name="c6284"></a>C6284
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

aviso C6284: objeto passado como parâmetro '% d' quando a cadeia de caracteres é necessária na chamada para \<function> .  
  
 Esse aviso indica que a cadeia de caracteres de formato especifica uma cadeia de caracteres, por exemplo, uma `%s` especificação para `printf` ou `scanf` , mas um objeto C++ foi passado em vez disso.  
  
 Esse defeito pode produzir saída ou falhas incorretas.  
  
 Essa mensagem geralmente é relatada devido à passagem de um objeto C++ que implementa algum tipo de cadeia de caracteres, por exemplo, `std::string` , `CComBSTR` ou `bstr_t` , em uma `printf` chamada em estilo C. Dependendo da implementação da classe C++, ou seja, se os operadores de conversão apropriados forem definidos, os objetos de cadeia de caracteres C++ geralmente poderão ser usados de forma transparente sempre que as cadeias de caracteres C forem necessárias; no entanto, como os parâmetros para `printf` funções de estilo são essencialmente sem tipo, nenhuma conversão em uma cadeia de caracteres ocorre.  
  
 Dependendo do objeto, pode ser apropriado inserir um operador no tipo de `static_cast` cadeia de caracteres apropriado, por exemplo, `char *` ou `TCHAR``*` , ou para chamar uma função de membro que retorna uma cadeia de caracteres, por exemplo, `c_str()` , em instâncias do `std::string` .  
  
## <a name="example"></a>Exemplo  
 O código a seguir gera esse aviso porque um `CComBSTR` é passado para a `sprintf` função:  
  
```  
#include <atlbase.h>  
#include <stdlib.h>  
  
void f()  
{  
  char buff[50];  
  CComBSTR bstrValue("Bye");  
  
  sprintf(buff,"%ws",bstrValue);   
}  
```  
  
 O código a seguir usa a conversão estática para corrigir este aviso:  
  
```  
#include <atlbase.h>  
#include <stdlib.h>  
  
void f()  
{  
  char buff[50];  
  CComBSTR bstrValue("Bye");  
  
  sprintf_s(buff,50,"%ws",static_cast<wchar_t *>(bstrValue));  
}  
```
