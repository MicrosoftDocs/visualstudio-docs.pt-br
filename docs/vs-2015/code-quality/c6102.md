# <a name="c6102"></a>C6102
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

aviso C6102: usando \<variable> da chamada de função com falha na linha \<location> .  
  
 Esse aviso é relatado em vez de [C6001](../code-quality/c6001.md) quando uma variável não é definida porque foi marcada como um `_Out_` parâmetro em uma chamada de função anterior que falhou.  
  
 O problema pode ser que a função chamada anteriormente não tenha sido anotada completamente. Pode exigir `_Always_`, `_Outptr_result_nullonfailure_` (`_COM_Outptr_` para o código COM) ou uma anotação relacionada.
