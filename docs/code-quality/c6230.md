# <a name="c6230"></a>C6230

aviso C6230: conversão implícita entre tipos inteiros semanticamente diferentes: usando HRESULT em um contexto booliano

Esse aviso indica que um simples `HRESULT` é usado em um contexto em que um resultado booliano é esperado, como uma instrução `if`. Esse teste provavelmente produzirá resultados incorretos. Por exemplo, o valor de êxito típico para `HRESULT` (`S_OK`) é false quando é testado como um booliano.

## <a name="example"></a>Exemplo

O código a seguir gera este aviso:

```cpp
#include <windows.h>

VOID f( )
{
  LPMALLOC pMalloc;
  HRESULT hr = CoGetMalloc(1, &pMalloc);

  if (hr)
  {

    // code ...
  }
  else
  {
    // code ...
  }
}
```

Na maioria das situações, a macro [Succeeded](/windows/desktop/api/winerror/nf-winerror-succeeded) ou [Failed](/windows/desktop/api/winerror/nf-winerror-failed) deve ser usada para testar o valor do `HRESULT`. Para corrigir esse aviso, use o seguinte código:

```cpp
#include <windows.h>

VOID f( )
{
  LPMALLOC pMalloc;
  HRESULT hr = CoGetMalloc(1, &pMalloc);

  if (SUCCEEDED(hr))
  {

    // code ...
  }
  else
  {
    // code ...
  }
}
```

Para esse aviso, o tipo `SCODE` é tratado como um `HRESULT`.

O uso de `malloc` e `free` (e APIs de memória dinâmica relacionadas) tem muitas armadilhas como uma causa de vazamentos e exceções de memória. Para evitar esses tipos de vazamentos e problemas de exceção, use as classes de ponteiro e contêiner fornecidas C++ pela biblioteca padrão. Isso inclui [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)e [vector](/cpp/standard-library/vector). Para obter mais informações, consulte [apontadores inteligentes](/cpp/cpp/smart-pointers-modern-cpp) e [ C++ biblioteca padrão](/cpp/standard-library/cpp-standard-library-reference).
