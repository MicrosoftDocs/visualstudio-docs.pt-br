# <a name="c6215"></a>C6215
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

aviso C6215: conversão entre tipos inteiros semanticamente diferentes: um tipo booliano para HRESULT  
  
 Esse aviso indica que um booliano está sendo convertido em um `HRESULT` . Os tipos boolianos indicam Success por um valor diferente de zero, enquanto Success ( `S_OK` ) in `HRESULT` é indicado por um valor de 0. Converter um tipo booliano em um `HRESULT` e usá-lo em uma expressão de teste gerará um resultado incorreto.  
  
 Esse aviso ocorre com frequência quando um booliano é usado como um argumento para `SUCCEEDED` ou `FAILED` macro, que converte explicitamente seus argumentos em um `HRESULT` .  
  
## <a name="example"></a>Exemplo  
 O código a seguir gera este aviso:  
  
```  
#include <windows.h>  
BOOL IsEqual(REFGUID, REFGUID);  
  
void f( REFGUID riid1, REFGUID riid2 )  
{  
  if (SUCCEEDED( IsEqual( riid1, riid2 ) )) //warning 6215  
  {  
    // success code ...  
  }  
  else  
  {  
    // failure code ...  
  }  
}  
```  
  
 Em geral, `SUCCEEDED` as `FAILED` macros ou só devem ser aplicadas ao `HRESULT` .  
  
 Para corrigir esse aviso, use o seguinte código:  
  
```  
#include <windows.h>  
BOOL IsEqual(REFGUID, REFGUID);  
  
void f( REFGUID riid1, REFGUID riid2 )  
{  
  if (IsEqual( riid1, riid2 ) == TRUE)   
  {  
    // code for riid1 == riid2  
  }  
  else  
  {  
    // code for riid1 != riid2  
  }  
}  
```  
  
 Para obter mais informações, consulte [macro com êxito](https://msdn.microsoft.com/library/ms687197.aspx) e [macro com falha](https://msdn.microsoft.com/library/ms693474.aspx)
