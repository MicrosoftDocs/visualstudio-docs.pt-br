# <a name="debugging-web-applications"></a>Depurando aplicativos Web

Este artigo fornece links para ajudá-lo a depurar diferentes tipos de aplicativos Web.

- Depurar aplicativos ASP.NET

  Para obter as etapas necessárias para habilitar a depuração de [!INCLUDE[vstecasp](../code-quality/includes/vstecasp_md.md)] aplicativos, consulte [depurar aplicativos ASP.net](how-to-enable-debugging-for-aspnet-applications.md).

- Depurar script do lado do cliente no ASP.NET

  Para obter as etapas necessárias, consulte a postagem do blog [Depurar JavaScript no Microsoft Edge](https://devblogs.microsoft.com/visualstudio/debug-javascript-in-microsoft-edge-from-visual-studio/) e esta [postagem para o Google Chrome](https://devblogs.microsoft.com/aspnet/client-side-debugging-of-asp-net-projects-in-google-chrome).

- Depurar JavaScript e TypeScript

  Consulte [Depurar JavaScript e TypeScript](../javascript/debug-nodejs.md).

- Exibir documentos de script

  Você pode ver listas de documentos de script do lado do servidor e do lado do cliente no **Gerenciador de soluções** para exibir. Você pode abrir qualquer documento de script do **Gerenciador de Soluções**. Para obter mais informações, consulte [How to: View script Documents](../debugger/how-to-view-script-documents.md).

- Depurando aplicativos de script AJAX

  Os aplicativos da Web com AJAX ativado fazem uso intenso de códigos de script e desafios especiais de depuração de representação. Para obter informações sobre técnicas de depuração AJAX, consulte [visão geral de depuração e rastreamento de aplicativos AJAX](/previous-versions/bb398817(v=vs.140)).
