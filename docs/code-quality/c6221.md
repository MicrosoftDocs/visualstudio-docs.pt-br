# <a name="c6221"></a>C6221
aviso C6221: Conversão implícita entre tipos inteiros semanticamente diferentes: comparando HRESULT a um inteiro. Considere alternativamente usar as macros SUCCEEDED ou FAILED

 Este aviso indica que um `HRESULT` está sendo comparado a um inteiro diferente de zero. Uma sucesso em `HRESULT` (`S_OK`) não é representada por 0. Portanto, uma conversão implícita de um `HRESULT` em um inteiro gerará um valor incorreto e provavelmente levará ao resultado incorreto. É geralmente causado equivocadamente aguardando uma função retornar um número inteiro quando ela na verdade retorna `HRESULT`.

## <a name="example"></a>Exemplo
 O código a seguir gera este aviso comparando `HRESULT` com um valor inteiro:

```cpp
#include <windows.h>

HRESULT f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  hr = CoGetMalloc(1, &pMalloc);
  if (hr == 4)
  {
    // failure code ...
    return S_FALSE;
  }
  else
  {
    // success code ...
    return S_OK;
  }
}
```

 Para corrigir este aviso, o código a seguir usa a macro `FAILED`:

```cpp
#include <windows.h>

HRESULT f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  hr = CoGetMalloc(1, &pMalloc);
  if (FAILED(hr))
  {
    // failure code ...
    return S_FALSE;
  }
  else
  {
    // success code ...
    return S_OK;
  }
}
```

 Para este aviso, o tipo `SCODE` é equivalente a `HRESULT`.

 Para obter mais informações, consulte [macro com êxito](/windows/win32/api/winerror/nf-winerror-succeeded) e [macro com falha](/windows/win32/api/winerror/nf-winerror-failed).

 Observe que o uso de malloc e free (e APIs de alocação de memória dinâmica relacionadas) têm várias armadilhas em termos de vazamentos de memória e exceções. Para evitar esses tipos de vazamentos e problemas de exceção no geral, use os mecanismos que são fornecidos pela STL (Biblioteca de Modelos Padrão) do C++. Isso inclui [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)e [vetor](/cpp/standard-library/vector). Para obter mais informações, consulte [apontadores inteligentes](/cpp/cpp/smart-pointers-modern-cpp) e [ C++ biblioteca padrão](/cpp/standard-library/cpp-standard-library-reference).
