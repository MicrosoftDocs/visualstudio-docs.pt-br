# <a name="c26402dont_heap_allocate_movable_result"></a>C26402 DONT_HEAP_ALLOCATE_MOVABLE_RESULT

Para evitar confusão sobre se um ponteiro possui um objeto, uma função que retorna um objeto móvel deve alocá-lo na pilha e retorná-lo por valor em vez de retornar um objeto alocado para heap. Se a semântica do ponteiro for necessária, retorne um ponteiro inteligente em vez de um ponteiro bruto. Consulte [ C++ as diretrizes básicas R. 3](https://github.com/isocpp/CppCoreGuidelines/blob/master/CppCoreGuidelines.md#Rr-ptr): *Warn se uma função retornar um objeto que foi alocado dentro da função, mas tiver um construtor de movimentação. Sugira considerar retorná-lo por valor.*
