# <a name="modifying-the-isolated-shell-by-using-the-pkgdef-file"></a>Modificar o Shell isolado usando o arquivo .Pkgdef
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

O arquivo. pkgdef dá suporte a configurações que você pode usar para personalizar um aplicativo de shell isolado. Ele especifica os valores que são criados quando um aplicativo é instalado em um computador e que são referenciados pelo shell do Visual Studio quando ele inicia o aplicativo. As configurações são organizadas no arquivo com base nas chaves do registro aplicáveis.  

> [!WARNING]
> Observe que os arquivos. pkgdef que não estão declarados no arquivo. vsixmanifest do VSPackage não são verificados quando o Visual Studio é iniciado.  

 O arquivo. pkgdef contém seções que são identificadas por uma chave, uma `[$RootKey$]` ou uma `[$RootKey$\` *subchave* `]` , em que $ROOTKEY $ é a chave raiz do aplicativo.  

 Cada seção contém pares de nome/valor que têm o seguinte formato: valueName `"` *ValueName* `"=` *valor*.  

 Os valores são uma cadeia de caracteres que é colocada entre aspas ou um inteiro de 32 bits que é representado como um DWORD. Os valores têm as seguintes restrições:  

- Todos os valores DWORD estão no formato hexadecimal, por exemplo `dword:00000001` .  

   Para valores Boolianos, 1 representa true e 0 representa false.  

- Todas as cadeias de caracteres de GUID estão no formato de registro, por exemplo, `"{00000000-0000-0000-0000-000000000000}"` .  

- Todos os identificadores de recursos localizáveis têm o formato "@*resourceName*" ou "#*ResourceId*", em que *ResourceId* é o identificador de recurso no pacote de interface do usuário do aplicativo, por exemplo, `"@102"` . O pacote de interface do usuário do aplicativo é o pacote que é referenciado na configuração AppLocalizationPackage.  

  Por exemplo,  

```  
"HideSolutionConcept"=dword:00000001  
"DefaultDebugEngine"="{00000000-0000-0000-0000-000000000000}"  
```  

 Você pode adicionar comentários ao arquivo. pkgdef. Um comentário de linha única tem duas barras como os dois primeiros caracteres.  

 Para obter uma lista das cadeias de caracteres de substituição, consulte [Strings de substituição usadas em. Pkgdef e. Arquivos Pkgundef](../extensibility/substitution-strings-used-in-dot-pkgdef-and-dot-pkgundef-files.md).  

 As seções a seguir descrevem valores específicos do registro que afetam o comportamento do shell do Visual Studio no modo isolado. Você também pode definir valores de registro adicionais para o aplicativo neste arquivo.  

> [!NOTE]
> Se uma configuração não for fornecida no arquivo. pkgdef, nenhuma entrada correspondente será feita no registro.

## <a name="binding-path-settings"></a>Configurações de caminho de associação  
 A chave [$RootKey $ \BindingPaths \\ {00000000-0000-0000-0000-000000000000} ] contém a lista de diretórios que o Shell verifica em busca de assemblies. Esses diretórios são adicionados à lista de diretórios que o Shell investiga para assemblies particulares para o aplicativo.  

 Por padrão, nenhuma entrada de caminho de associação é adicionada ao arquivo. pkgdef. No entanto, os seguintes subdiretórios do diretório de instalação do Visual Studio são adicionados automaticamente à lista de caminhos de associação de aplicativos no registro.  

- Common7\IDE\  

- Common7\IDE \\ \PrivateAssemblies  

- Common7\IDE \\ \PublicAssemblies  

  Para adicionar um diretório ao caminho de associação, adicione uma entrada no formato "*DirectoryName*" = "", em que *DirectoryName* é um caminho absoluto. Por exemplo,  

```  
[$RootKey$\BindingPaths\{00000000-0000-0000-0000-000000000000}]  
"$RootFolder$\directory1"=""  
"%CommonProgramFiles%\directory2"=""  
```

## <a name="package-menu-item-settings"></a>Configurações do item de menu do pacote  
 A chave do registro [$RootKey $ \Menus] define os arquivos de recursos da interface do usuário para o aplicativo.  

 Os valores de item de menu têm o formato "{*vsUiPackageGuid*}" = ", *ResourceId*, *versionNumber*", em que *vsUiPackageGuid* é o GUID do pacote de interface do usuário do aplicativo, *ResourceId* é o identificador de recurso do recurso CTMENU que contém os elementos da interface do usuário e *versionNumber* é um número de versão virtual para o recurso CTMENU. Para obter mais informações, consulte [Registrando manipuladores de comando de assembly de interoperabilidade](../extensibility/internals/registering-interop-assembly-command-handlers.md).  

 Por padrão, uma entrada de item de menu é criada no arquivo. pkgdef para o pacote de interface do usuário do aplicativo.  

 Para cada pacote que fornece itens de menu e que são distribuídos como parte do aplicativo, adicione uma entrada de item de menu para o pacote.
