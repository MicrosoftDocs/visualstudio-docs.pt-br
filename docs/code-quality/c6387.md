# <a name="c6387"></a>C6387
aviso C6387: o > de \<argument pode ser \<value >: isso não segue a especificação da função \<function nome >: linhas: x, y

 Esse aviso será gerado se um parâmetro de função anotada estiver sendo passado como um valor inesperado. Por exemplo, passar um valor potencialmente nulo para um parâmetro marcado com a anotação `_In_` gera esse aviso.

## <a name="example"></a>Exemplo
 O código a seguir gera esse aviso porque um parâmetro nulo é passado para `f(char *)`:

```cpp

#include <sal.h>

_Post_ _Null_ char * g();

void f(_In_ char *pch);

void main()
{
    char *pCh = g();
    f(pCh); // Warning C6387
}
```

 Para corrigir esse aviso, use o seguinte código:

```cpp

#include <sal.h>

_Post_ _Notnull_ char * g();

void f(_In_ char *pch);

void main()
{
    char *pCh = g();
    f(pCh);
}
```
