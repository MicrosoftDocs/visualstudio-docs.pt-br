# <a name="sdk-reference-packages"></a>Pacotes de referência do SDK

[!INCLUDE [preview-note](../includes/preview-note.md)]

A maneira mais fácil de criar extensões do Visual Studio é com uma referência ao [ `Microsoft.VisualStudio.Sdk` pacote NuGet](https://www.nuget.org/packages/microsoft.visualstudio.sdk).
Este pacote está disponível para o direcionamento do Visual Studio 2017 (15,0), do Visual Studio 2019 (16,0, 16,9) e agora do Visual Studio 2022.

Dependendo da extensão, adicionar pacotes adicionais do SDK do VS que não estão incluídos no meta-pacote acima pode ser necessário.
Ao fazer referência a outros pacotes SDK específicos, esses pacotes podem variar em versões principais do VS.

Observe que muitos assemblies de interoperabilidade eram incorporáveis antes do Visual Studio 2022. A partir do Visual Studio 2022, a inserção não é mais necessária ou tem suporte.
*Consulte* nossos assemblies de interoperabilidade em vez de vinculá-los.

A tabela a seguir fornece um mapeamento de assemblies ou pacotes que sua extensão anterior ao Visual Studio 2022 pode já estar fazendo referência à nova ID de pacote a ser referenciada ao direcionar o Visual Studio 2022. Em alguns casos, os assemblies agora estão disponíveis em pacotes NuGet que anteriormente só estavam disponíveis em uma instalação local do Visual Studio.
