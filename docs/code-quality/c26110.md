# <a name="c26110"></a>C26110
C26110 de aviso: O chamador está falhando em manter o bloqueio \<lock > antes de chamar a função \<func >.

 Quando um bloqueio é necessário, certifique-se de esclarecer se a própria função ou seu chamador deve adquirir o bloqueio. Aviso C26110 é emitido quando há uma violação da anotação `_Requires_lock_held_` ou outras anotações relacionadas ao bloqueio. Para obter mais informações, consulte [anotando o comportamento de bloqueio](annotating-locking-behavior.md)

## <a name="example"></a>Exemplo
 No exemplo a seguir, o aviso C26110 é gerado porque a anotação `_Requires_lock_held_` na função `LockRequired` indica que o chamador de `LockRequired` deve adquirir o bloqueio antes de chamar `LockRequired`. Sem essa anotação, `LockRequired` precisa adquirir o bloqueio antes de acessar todos os dados compartilhados protegidos pelo bloqueio.

```

typedef struct _DATA
{
    CRITICAL_SECTION cs;
    int d;
} DATA;

_Requires_lock_held_(p->cs)

void LockRequired(DATA* p)
{
    p->d = 0;
}

void LockNotHeld(DATA* p)
{
    LockRequired(p); // Warning C26110
}
```
