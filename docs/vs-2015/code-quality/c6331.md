# <a name="c6331"></a>C6331
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Aviso C6331: Parâmetro inválido: passar MEM_RELEASE e MEM_DECOMMIT em conjunto para \<função > não é permitido. Isso resulta na falha dessa chamada  
  
 Esta mensagem indica que um parâmetro inválido, que está sendo passado para VirtualFree ou VirtualFreeEx. VirtualFree e VirtualFreeEx rejeitam os sinalizadores (MEM_RELEASE &#124; MEM_DECOMMIT) em combinação. Portanto, os valores MEM_DECOMMIT e MEM_RELEASE podem não ser usados juntos na mesma chamada.  
  
 Não é necessário para desfazer a confirmação e liberar para ocorrer como etapas independentes. Liberar a memória confirmada desconfirmará as páginas também. Além disso, verifique se que o valor de retorno dessa função não é ignorado.  
  
## <a name="example"></a>Exemplo  
 O código de exemplo a seguir gera este aviso:  
  
```  
#include <windows.h>  
#define PAGELIMIT 80  
  
DWORD dwPages = 0;  // count of pages   
DWORD dwPageSize;   // page size   
  
VOID fd( VOID )  
{  
  LPVOID lpvBase;            // base address of the test memory  
  BOOL bSuccess;             
  SYSTEM_INFO sSysInfo;      // system information  
  
  GetSystemInfo( &sSysInfo );    
  dwPageSize = sSysInfo.dwPageSize;  
  
  // Reserve pages in the process's virtual address space  
  lpvBase = VirtualAlloc (  
                       NULL,                 // system selects address  
                       PAGELIMIT*dwPageSize, // size of allocation  
                       MEM_RESERVE,          
                       PAGE_NOACCESS );       
  if (lpvBase)  
  {  
    // code to access memory   
  }  
  else  
  {  
    return;  
  }  
  bSuccess = VirtualFree(lpvBase,              
                0,  
                MEM_DECOMMIT | MEM_RELEASE); // warning   
  // code...  
}  
```  
  
 Para corrigir este aviso, passe o valor MEM_DECOMMIT para a chamada VirtualFree, conforme mostrado no código a seguir:  
  
```  
#include <windows.h>  
#define PAGELIMIT 80  
  
DWORD dwPages = 0;  // count of pages   
DWORD dwPageSize;   // page size   
  
VOID f( VOID )  
{  
  LPVOID lpvBase;            // base address of the test memory  
  BOOL bSuccess;             
  SYSTEM_INFO sSysInfo;      // system information  
  
  GetSystemInfo( &sSysInfo );    
  dwPageSize = sSysInfo.dwPageSize;  
  
  // Reserve pages in the process's virtual address space  
  lpvBase = VirtualAlloc (  
                       NULL,                 // system selects address  
                       PAGELIMIT*dwPageSize, // size of allocation  
                       MEM_RESERVE,          
                       PAGE_NOACCESS );       
  if (lpvBase)  
  {  
    // code to access memory   
  }  
  else  
  {  
    return;  
  }  
  bSuccess = VirtualFree(lpvBase, 0, MEM_RELEASE);   
  // code...  
}  
```  
  
 Observe que o uso de malloc e free (e APIs de alocação de memória dinâmica relacionada) têm várias armadilhas em termos de vazamentos de memória e exceções. Para evitar esses tipos de vazamentos e totalmente de problemas de exceção, use os mecanismos que são fornecidos pela biblioteca padrão de modelo C++ (STL). Eles incluem [shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860), [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a), e [vetor](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f). Para obter mais informações, consulte [ponteiros inteligentes](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435) e [biblioteca padrão C++](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee).
