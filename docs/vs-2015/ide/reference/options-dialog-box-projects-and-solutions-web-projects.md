# <a name="options-dialog-box-projects-and-solutions-web-projects"></a>Caixa de diálogo Opções, Projetos e Soluções, Projetos Web
[!INCLUDE[vs2017banner](../../includes/vs2017banner.md)]

Define o servidor Web que projetos Web usarão para desenvolvimento dentro do [!INCLUDE[vsprvs](../../includes/vsprvs-md.md)]. Para acessar essa caixa de diálogo, clique em **Opções de Ferramentas**. Expanda **Projetos e Soluções** e clique em **Projetos Web**.

 Por padrão, quando você executa um projeto Web no Visual Studio (por exemplo, usando F5 ou Ctrl+F5), o Visual Studio usa o Visual Studio Development Server. Para obter mais informações, consulte [Servidores Web no Visual Studio para projetos Web ASP.NET](https://msdn.microsoft.com/31d4f588-df59-4b7e-b9ea-e1f2dd204328).

> [!NOTE]
> As opções disponíveis nas caixas de diálogo e os nomes os locais dos comandos de menu que você vê podem diferir do que é descrito na Ajuda, dependendo de suas configurações ativas ou da edição. Esta página de Ajuda foi escrita pensando nas **Configurações da Web** em mente. Para exibir ou alterar as configurações, escolha **Importar e Exportar Configurações** no menu **Ferramentas**. Para obter mais informações, consulte [Personalizando configurações de desenvolvimento no Visual Studio](https://msdn.microsoft.com/22c4debb-4e31-47a8-8f19-16f328d7dcd3).

## <a name="settings"></a>Configurações
 **Usar a versão de 64 bits do IIS Express para projetos e sites da Web** Selecione esta opção para usar IIS Express em vez do Visual Studio Development Server. Para obter mais informações, consulte [Apresentando o IIS Express](https://weblogs.asp.net/scottgu/introducing-iis-express) e [visão geral do IIS Express](/iis/extensions/introduction-to-iis-express/iis-express-overview). Essa opção é desabilitada por padrão.

 **Avisar antes de executar aplicativos Web quando houver erros na lista de erros** Se essa caixa estiver marcada, você será avisado se tentar executar seu aplicativo Web quando ele não for compilado sem erros.
