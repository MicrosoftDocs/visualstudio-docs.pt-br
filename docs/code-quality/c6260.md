# <a name="c6260"></a>C6260
aviso C6260: sizeof * sizeof está quase sempre errado, você pretendia usar uma contagem de caracteres ou uma contagem de bytes?

 Esse aviso indica que os resultados de duas operações `sizeof` foram multiplicados juntos. O operador CC++ /`sizeof` retorna o número de bytes de armazenamento que um objeto usa. Normalmente, ele é incorreto para multiplicar isso por outra operação `sizeof`; Geralmente, um está interessado no número de bytes em um objeto ou no número de elementos em uma matriz (por exemplo, o número de caracteres largos em uma matriz).

 Há um comportamento não intuitivo associado ao operador `sizeof`. Por exemplo, em C, o `sizeof ('\0') == 4,` porque um caractere é de um tipo integral. No C++, o tipo de um literal de caractere é `char`, portanto, `sizeof ('\0') == 1`. No entanto, em C C++e, o seguinte é verdadeiro:

```
sizeof ("\0") == 2.
```

## <a name="example"></a>Exemplo
 O código a seguir gera este aviso:

```
#include <windows.h>

void f( )
{
  int i;
  i = sizeof (L"String") * sizeof (WCHAR);
  // code ...
}
```

 Para corrigir esse aviso, use o seguinte código:

```
#include <windows.h>

void f( )
{
  int i;
  i= sizeof (L"String") / sizeof (WCHAR);

  /* or to get bytes */
  i = sizeof (L"String");
  // code ...
}
```
