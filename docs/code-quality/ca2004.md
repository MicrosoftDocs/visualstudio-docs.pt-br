# <a name="ca2004-remove-calls-to-gckeepalive"></a>CA2004: remover chamadas para GC.KeepAlive

|||
|-|-|
|NomeDoTipo|RemoveCallsToGCKeepAlive|
|CheckId|CA2004|
|Categoria|Microsoft.Reliability|
|Alteração significativa|Sem interrupção|

## <a name="cause"></a>Causa
As classes usam `SafeHandle`, mas ainda contêm chamadas para `GC.KeepAlive`.

## <a name="rule-description"></a>Descrição da regra
Se você estiver convertendo para `SafeHandle` uso, remova todas as chamadas para `GC.KeepAlive` (objeto). Nesse caso, as classes não devem chamar `GC.KeepAlive`, supondo que elas não tenham um finalizador, mas dependem de `SafeHandle` para concluir o identificador do sistema operacional para elas.  Embora o custo de deixar em uma chamada para `GC.KeepAlive` possa ser insignificante como medido pelo desempenho, a percepção de que uma chamada para `GC.KeepAlive` é necessária ou suficiente para resolver um problema de tempo de vida que pode não existir mais para que o código seja mais difícil de manter.

## <a name="how-to-fix-violations"></a>Como corrigir violações
Remova as chamadas para `GC.KeepAlive`.

## <a name="when-to-suppress-warnings"></a>Quando suprimir avisos
Você pode suprimir esse aviso somente se não estiver tecnicamente correto para converter para `SafeHandle` uso em sua classe.
