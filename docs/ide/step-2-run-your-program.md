# <a name="step-2-run-your-picture-viewer-app"></a>Passo 2: Execute seu aplicativo de visualizador de imagens

Quando você cria um projeto do Aplicativo Formulários do Windows, você realmente constrói um programa que é executado. Neste tutorial, seu aplicativo de visualização&mdash;de imagens ainda não faz muito, embora o faça. Por enquanto, ele exibe uma janela vazia que mostra **o Form1** na barra de título.

Veja como executar seu aplicativo. 

1. Escolha um dos seguintes métodos:

    - Pressione a tecla **F5**.

    - Na barra de menu, escolha **Depurar** > **depuração**.

    - Na barra de ferramentas, escolha o botão **Iniciar depuração,** que aparece da seguinte forma:

      ![Botão de barra de ferramentas Iniciar Depuração](../ide/media/express_icondebug.png)<br>
      Iniciar botão de barra de *ferramentas* ***de depuração***

1. O Visual Studio executa seu aplicativo, e uma janela chamada **Form1** aparece. A captura de tela a seguir mostra o aplicativo que você acabou de construir. O aplicativo está sendo executado, e você logo adicionará a ele.

     ![Aplicativo Do Windows Forms em execução](../ide/media/express_firstrun.png)<br>
***Aplicativo de formulários do Windows***, *em execução*

1. Volte para o ambiente de desenvolvimento integrado do Visual Studio (IDE) e, em seguida, olhe para a nova barra de ferramentas. Botões adicionais aparecem na barra de ferramentas quando você executa um aplicativo. Esses botões permitem que você faça coisas como parar e iniciar seu aplicativo, e ajudá-lo a rastrear quaisquer erros (bugs) que ele possa ter. Para este exemplo, estamos usando-o para iniciar e parar o aplicativo.

     ![Barra de ferramentas de depuração](../ide/media/express_debugtoolbar.png)<br>
Barra de *ferramentas* ***de depuração***

1. Use um dos seguintes métodos para parar seu aplicativo:

    - Na barra de ferramentas, clique no botão **Parar Depuração**.

    - Na barra de menu, escolha **Depurar** > **Parar de Depurar**.

    - Use o teclado e **pressione Shift**+**F5**.

    - Clique no botão **X** no canto superior da janela **Form1**.

    > [!NOTE]
    > Quando você executa seu aplicativo de dentro do IDE, ele é chamado de depuração porque você normalmente faz isso para localizar e corrigir bugs (erros) no aplicativo. Embora este aplicativo seja pequeno e não faça muito ainda, ainda é um programa real. Você segue o mesmo procedimento para executar e depurar outros programas. Para saber mais sobre depuração, confira [Introdução ao depurador](../debugger/debugger-feature-tour.md).
