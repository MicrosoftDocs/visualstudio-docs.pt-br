# <a name="c6216"></a>C6216
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

aviso C6216: conversão inserida pelo compilador entre tipos integrais semanticamente diferentes: um tipo booliano para HRESULT  
  
 Esse aviso indica que um booliano está sendo usado como um `HRESULT` sem conversão explícita. Tipos boolianos indicam sucesso por um valor diferente de zero; Success ( `S_OK` ) em `HRESULT` é indicado por um valor de 0. O valor de falha típico para funções que retornam um booliano falso é um status de êxito quando é testado como um `HRESULT` . Isso provavelmente levará a resultados incorretos.  
  
## <a name="example"></a>Exemplo  
 O código a seguir gera este aviso:  
  
```  
#include <windows.h>  
BOOL IsEqual(REFGUID, REFGUID);  
  
HRESULT f( REFGUID riid1, REFGUID riid2 )  
{  
  // code ...  
  return IsEqual(riid1, riid2);    
}  
```  
  
 Para corrigir esse aviso, use o seguinte código:  
  
```  
#include <windows.h>  
BOOL IsEqual(REFGUID, REFGUID);  
  
HRESULT f( REFGUID riid1, REFGUID riid2 )  
{  
  if (IsEqual(riid1, riid2) == TRUE)  
  {  
    // code ...  
    return S_OK;  
  }  
  else  
  {  
    // code ...  
    return E_FAIL;  
  }  
}  
```  
  
 Para esse aviso, o `SCODE` tipo é equivalente a `HRESULT` .  
  
 Para obter mais informações, consulte [macro com êxito](https://msdn.microsoft.com/library/ms687197.aspx) e [macro com falha](https://msdn.microsoft.com/library/ms693474.aspx).
