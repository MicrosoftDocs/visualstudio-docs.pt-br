# <a name="c6250"></a>C6250
aviso C6250: a chamada a \<function > VirtualFree sem o sinalizador MEM_RELEASE pode liberar memória, mas não descritores de endereço (VADs); resulta em vazamentos de espaço de endereço

 Esse aviso indica que uma chamada para `VirtualFree` sem o sinalizador `MEM_RELEASE` apenas deconfirma as páginas e não as libera. Para desconfirmar e liberar páginas, use o sinalizador `MEM_RELEASE` na chamada para `VirtualFree`. Se alguma página na região for confirmada, a função primeiro será desconfirmada e, em seguida, as liberará. Após essa operação, as páginas estão no estado livre. Se você especificar esse sinalizador, `dwSize` deverá ser zero e `lpAddress` deverá apontar para o endereço base retornado pela função `VirtualAlloc` quando a região tiver sido reservada. A função falhará se nenhuma dessas condições for atendida.

 Você pode ignorar esse aviso se o código liberar mais tarde o espaço de endereço chamando `VirtualFree` com o sinalizador `MEM_RELEASE`.

 Para obter mais informações, consulte [VirtualAlloc](http://go.microsoft.com/fwlink/?LinkId=181201) e [VirtualFree](http://go.microsoft.com/fwlink/?LinkId=181202).

 O uso de VirtualAlloc e VirtualFree tem muitas armadilhas em termos de vazamentos de memória e exceções. Para evitar esses tipos de vazamentos e problemas de exceção totalmente, use os mecanismos fornecidos pela STL ( C++ Standard Template Library). Isso inclui [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)e [vector](/cpp/standard-library/vector). Para obter mais informações, consulte [apontadores inteligentes](/cpp/cpp/smart-pointers-modern-cpp) e [ C++ biblioteca padrão](/cpp/standard-library/cpp-standard-library-reference).

## <a name="example"></a>Exemplo
 O código de exemplo a seguir gera este aviso:

```
#include <windows.h>
#include <stdio.h>
#define PAGELIMIT 80

DWORD dwPages = 0;  // count of pages
DWORD dwPageSize;   // page size

VOID f( )
{
  LPVOID lpvBase;            // base address of the test memory
  SYSTEM_INFO sSysInfo;      // system information

  GetSystemInfo(&sSysInfo);
  dwPageSize = sSysInfo.dwPageSize;

  // Reserve pages in the process's virtual address space
  lpvBase = VirtualAlloc(
                         NULL,                // system selects address
                         PAGELIMIT*dwPageSize, // size of allocation
                         MEM_RESERVE,
                         PAGE_NOACCESS);
  //
  // code to access memory
  // ...

  if (lpvBase != NULL)
  {
    if (VirtualFree( lpvBase, 0, MEM_DECOMMIT )) // decommit pages
      {
        puts ("MEM_DECOMMIT Succeeded");
      }
      else
      {
       puts("MEM_DECOMMIT failed");
      }
  }
  else
  {
    puts("lpvBase == NULL");
  }
}
```

 Para corrigir esse aviso, use o seguinte código de exemplo:

```
#include <windows.h>
#include <stdio.h>
#define PAGELIMIT 80

DWORD dwPages = 0;  // count of pages
DWORD dwPageSize;   // page size

VOID f( )
{
  LPVOID lpvBase;            // base address of the test memory
  SYSTEM_INFO sSysInfo;      // system information

  GetSystemInfo(&sSysInfo);
  dwPageSize = sSysInfo.dwPageSize;

  // Reserve pages in the process's virtual address space
  lpvBase = VirtualAlloc(
                         NULL,                // system selects address
                         PAGELIMIT*dwPageSize, // size of allocation
                         MEM_RESERVE,
                         PAGE_NOACCESS);
  //
  // code to access memory
  // ...

  if (lpvBase != NULL)
  {
   if (VirtualFree(lpvBase, 0,MEM_RELEASE )) // decommit & release
   {
     // code ...
   }
   else
   {
     puts("MEM_RELEASE failed");
   }
  }
  else
  {
    puts("lpvBase == Null ");
    // code...
  }
}
```
