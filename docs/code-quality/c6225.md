# <a name="c6225"></a>C6225

C6225 de aviso: Conversão implícita entre tipos inteiros semanticamente diferentes: atribuindo 1 ou TRUE a HRESULT. Considere usar S_FALSE em vez disso

Esse aviso indica que um `HRESULT` está sendo atribuído ou inicializado com um valor de um 1 explícito. Tipos boolianos indicam sucesso por um valor diferente de zero; Success (`S_OK`) em `HRESULT` é indicado por um valor de 0. Esse aviso é frequentemente causado por uma confusão acidental dos tipos Boolean e `HRESULT`. Para indicar o êxito, a constante simbólica `S_OK` deve ser usada.

## <a name="example"></a>Exemplo

No código a seguir, a atribuição de `HRESULT` gera este aviso:

```cpp
#include <windows.h>

VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (SUCCEEDED(CoGetMalloc(1, &pMalloc)))
  {
    // code ...
    hr = S_OK;
  }
  else
  {
    // code ...
    hr = 1;
  }
}
```

Para corrigir esse aviso, use o seguinte código:

```cpp
VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (SUCCEEDED(CoGetMalloc(1, &pMalloc)))
  {
    hr = S_OK;
    // code ...
  }
  else
  {
    hr = S_FALSE;
    // code ...
  }
}
```

Para esse aviso, o tipo `SCODE` é equivalente a `HRESULT`.

Para indicar falha, `E_FAIL` ou outra constante deve ser usada.
