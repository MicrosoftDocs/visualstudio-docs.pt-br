# <a name="what39s-new-in-the-source-control-plug-in-api-version-12"></a>O que&#39;é novidade na API plug-in de controle de fonte versão 1.2
Esta seção destaca a funcionalidade avançada de controle de código-fonte na versão 1.2 do Plug-in de controle de fonte. A seguir lista os novos recursos na versão 1.2:

## <a name="changes"></a>Alterações
- [Eliminação de arquivos ~SAK](../../extensibility/internals/elimination-of-tilde-sak-files.md) Explica como a API 1.2 do Source Control Plug-in interage com arquivos temporários.

- [Aplicação de configurações em várias conexões de projeto](../../extensibility/internals/application-of-settings-across-multiple-project-connections.md) Ilustra como um plug-in de controle de origem pode usar uma operação em lote para executar a mesma operação de controle de origem em vários projetos ou contextos de conexão múltiplos.

- [Criando pastas de contêineres-pais para soluções](../../extensibility/internals/creating-parent-container-folders-for-solutions.md) Explica como a API Plug-in de controle de origem versão 1.2 permite que um usuário especifique um único destino de controle de origem raiz para todos os projetos da Web dentro da solução.

- [Comparação opcional da pasta de projeto local para a loja de controle de origem](../../extensibility/internals/optional-comparison-of-local-project-folder-to-source-control-store.md) Explica como a API 1.2 do Source Control Plug-in compara versões de arquivos.

- [Remoção de informações de controle de origem de . Proj e . Arquivos SLn](../../extensibility/internals/removal-of-source-control-information-from-dot-proj-and-dot-sln-files.md) Explica como a API Plug-in de Controle de Fonte Versão 1.2 armazena informações sobre projetos e soluções.
