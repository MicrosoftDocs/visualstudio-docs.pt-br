# <a name="c6332"></a>C6332
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Aviso C6332: parâmetro inválido: passando zero como o parâmetro dwFreeType para \<function> não é permitido. Isso resulta na falha dessa chamada  
  
 Esse aviso indica que um parâmetro inválido está sendo passado para VirtualFree ou VirtualFreeEx. VirtualFree e VirtualFreeEx rejeitam um parâmetro dwFreeType de zero. O parâmetro dwFreeType pode ser MEM_DECOMMIT ou MEM_RELEASE. No entanto, os valores MEM_DECOMMIT e MEM_RELEASE não podem ser usados juntos na mesma chamada. Além disso, certifique-se de que o valor de retorno da função VirtualFree não seja ignorado.  
  
## <a name="example"></a>Exemplo  
 O código a seguir gera esse aviso porque um parâmetro inválido é passado para a função VirtualFree:  
  
```  
#include <windows.h>  
#define PAGELIMIT 80              
  
DWORD dwPages = 0;  // count of pages   
DWORD dwPageSize;   // page size   
  
VOID f( VOID )  
{  
  LPVOID lpvBase;            // base address of the test memory  
  BOOL bSuccess;             
  SYSTEM_INFO sSysInfo;      // system information  
  
  GetSystemInfo( &sSysInfo );    
  dwPageSize = sSysInfo.dwPageSize;  
  
  // Reserve pages in the process's virtual address space  
  lpvBase = VirtualAlloc(  
                         NULL,                // system selects address  
                         PAGELIMIT*dwPageSize, // size of allocation  
                         MEM_RESERVE,          
                         PAGE_NOACCESS );       
  if (lpvBase)  
  {  
    // code to access memory   
  }  
  else  
  {  
    return;  
  }  
  
  bSuccess = VirtualFree( lpvBase, 0, 0 );   
  // code ...  
}  
```  
  
 Para corrigir esse aviso, modifique a chamada para a função VirtualFree, conforme mostrado no código a seguir:  
  
```  
#include <windows.h>  
#define PAGELIMIT 80              
  
DWORD dwPages = 0;  // count of pages   
DWORD dwPageSize;   // page size   
  
VOID f( VOID )  
{  
  LPVOID lpvBase;            // base address of the test memory  
  BOOL bSuccess;             
  SYSTEM_INFO sSysInfo;      // system information  
  
  GetSystemInfo( &sSysInfo );    
  dwPageSize = sSysInfo.dwPageSize;  
  
  // Reserve pages in the process's virtual address space  
  lpvBase = VirtualAlloc(  
                         NULL,                // system selects address  
                         PAGELIMIT*dwPageSize, // size of allocation  
                         MEM_RESERVE,          
                         PAGE_NOACCESS );       
  if (lpvBase)  
  {  
    // code to access memory   
  }  
  else  
  {  
    return;  
  }  
  
  bSuccess = VirtualFree( lpvBase, 0, MEM_RELEASE );  
  // code ...  
}   
```  
  
 O uso de VirtualAlloc e VirtualFree tem muitas armadilhas em termos de vazamentos de memória e exceções. Para evitar esses tipos de vazamentos e problemas de exceção totalmente, use os mecanismos fornecidos pela STL (biblioteca de modelos padrão) do C++. Isso inclui [shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860), [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)e [vetor](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f). Para obter mais informações, consulte [apontadores inteligentes](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435) e a [biblioteca padrão C++](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee).
