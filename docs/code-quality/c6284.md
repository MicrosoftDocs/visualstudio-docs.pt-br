# <a name="c6284"></a>C6284
aviso C6284: objeto passado como parâmetro '% d' quando a cadeia de caracteres é necessária na chamada de \<function >.

 Esse aviso indica que a cadeia de caracteres de formato especifica uma cadeia de caracteres, por exemplo, uma especificação `%s` para `printf` ou @no__t C++ -2, mas um objeto foi passado em vez disso.

 Esse defeito pode produzir saída ou falhas incorretas.

 Geralmente, essa mensagem é relatada devido C++ à passagem de um objeto que implementa algum tipo de cadeia de caracteres, por exemplo, `std::string`, `CComBSTR` ou `bstr_t`, em uma chamada de estilo-4 C @no__t. Dependendo da implementação da C++ classe, ou seja, se os operadores de conversão apropriados forem definidos, os objetos C++ de cadeia de caracteres geralmente poderão ser usados de forma transparente sempre que as cadeias de caracteres C forem necessárias; no entanto, como os parâmetros para @no__t funções de estilo -2 são essencialmente sem tipo, nenhuma conversão em uma cadeia de caracteres ocorre.

 Dependendo do objeto, pode ser apropriado inserir um operador `static_cast` no tipo de cadeia de caracteres apropriado, por exemplo, `char *` ou `TCHAR``*` ou chamar uma função de membro que retorna uma cadeia de caracteres, por exemplo, `c_str()`, em instâncias de `std::string`.

## <a name="example"></a>Exemplo
 O código a seguir gera esse aviso porque um `CComBSTR` é passado para a função `sprintf`:

```
#include <atlbase.h>
#include <stdlib.h>

void f()
{
  char buff[50];
  CComBSTR bstrValue("Bye");

  sprintf(buff,"%ws",bstrValue);
}
```

 O código a seguir usa a conversão estática para corrigir este aviso:

```
#include <atlbase.h>
#include <stdlib.h>

void f()
{
  char buff[50];
  CComBSTR bstrValue("Bye");

  sprintf_s(buff,50,"%ws",static_cast<wchar_t *>(bstrValue));
}
```
